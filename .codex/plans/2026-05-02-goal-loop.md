# Accepted Plan: Goal-Based `codex-loop` Condition

Accepted on 2026-05-02 after the user requested implementation.

## Summary

Add `goal` as the third exclusive loop condition, alongside `min` and `rounds`.

A goal loop is activated with:

```text
[[CODEX_LOOP name="<name>" goal="<optional extra goal>" confirm_model="<optional model>" confirm_reasoning_effort="<optional effort>"]]
<prompt>
```

The loop continues until a headless `codex exec` confirmation pass says the original task is actually complete. The optional `goal` string supplements the task prompt; `goal=""` is valid and means the prompt itself is the goal.

By default, goal confirmation uses `gpt-5.5` with `model_reasoning_effort = "high"`.

## Key Changes

- Activation and state:
  - Add `LimitModeGoal = "goal"`.
  - Require exactly one limiter among `min`, `rounds`, and `goal`; `goal=""` counts as present.
  - Allow `confirm_model` and `confirm_reasoning_effort` only with `goal`.
  - Store model and reasoning separately: `confirm_model = "gpt-5.5"` and `confirm_reasoning_effort = "high"` by default.
  - Do not support combined pseudo-model strings like `gpt-5.5-xhigh`; reasoning is a separate Codex config value.
  - Extend loop state with goal-specific fields: `goal_text`, `confirm_model`, `confirm_reasoning_effort`, `goal_check_count`, and last goal verdict metadata.
  - Bump the loop record version while keeping existing `time` and `rounds` records readable.
- Runtime config:
  - Add `[hooks].stop_timeout_seconds = 2700`.
  - Add `[goal] confirm_model = "gpt-5.5"`, `confirm_reasoning_effort = "high"`, `codex_command = "codex"`, `timeout_seconds = 2400`, and `max_output_bytes = 12000`.
  - `hooks.stop_timeout_seconds` controls the managed Codex `Stop` hook timeout written into `~/.codex/hooks.json`.
  - `goal.timeout_seconds` controls the nested `codex exec` confirmation timeout and must leave a grace window before the outer hook timeout.
  - Resolve confirmation model in this order: header `confirm_model`, non-empty `[goal].confirm_model`, Stop payload `model` when present, then Codex default by omitting `--model`.
  - Resolve confirmation reasoning effort in this order: header `confirm_reasoning_effort`, non-empty `[goal].confirm_reasoning_effort`, then Codex default by omitting the config override.
  - Accept only documented reasoning efforts: `minimal`, `low`, `medium`, `high`, `xhigh`.
- Hook installation and timeout behavior:
  - Change the managed `Stop` hook timeout from the current fixed `30` seconds to configurable `hooks.stop_timeout_seconds`.
  - Keep `UserPromptSubmit` timeout at `30` seconds.
  - Make `codex-loop install` read runtime config before rendering managed hooks.
  - Document that changing `hooks.stop_timeout_seconds` requires rerunning `codex-loop install` and restarting/reloading Codex.
  - Update bundled `plugins/codex-loop/hooks/hooks.json` to use the 45-minute default.
- Goal confirmation runner:
  - Run `codex exec` headlessly from the resolved workspace root with `--cd`, `--ephemeral`, `--sandbox read-only`, `--ask-for-approval never`, `--output-schema`, and `--output-last-message`.
  - With defaults, pass `--model gpt-5.5` and `--config model_reasoning_effort="high"`.
  - Feed a reviewer prompt that does not start with `[[CODEX_LOOP`, includes the original task prompt, optional goal text, latest assistant message, loop metadata, and strict instructions to inspect only and return schema-valid JSON.
  - Expected confirmation JSON shape: `completed`, `confidence`, `reason`, `missing_work`, and `next_round_guidance`.
  - Treat invalid JSON, schema failure, non-zero exit, missing `codex`, timeout, or cancellation as not confirmed complete.
- Stop hook flow:
  - For `goal` mode, run the goal confirmation after updating `last_assistant_message` and `last_stop_at`, before `pre_loop_continue`.
  - If confirmation returns `completed: true`, mark the loop `completed`, write state, append a JSONL event, and return no hook output.
  - If confirmation returns incomplete or fails, continue the loop unless existing rapid-stop protection cuts it short.
  - Run `pre_loop_continue` only after the goal decision says another round is needed.
  - Keep `Stop` stdout as one valid JSON object; goal confirmation and `pre_loop_continue` content are appended as separate labeled sections inside the `reason`.
- JSONL observability:
  - Add `~/.codex/codex-loop/runs.jsonl`.
  - Append one `goal_check` event per goal confirmation attempt.
  - Use append-only local JSONL with mode `0600`.
  - Store verdict metadata only; do not log full task prompt, full assistant message, full confirm prompt, or `pre_loop_continue` stdout by default.

## Test Plan

- Parser and activation tests for goal presence, exclusivity, `confirm_model`, and `confirm_reasoning_effort`.
- Config/default tests for `gpt-5.5`, `high`, timeout normalization, and separate subprocess args.
- Store/status tests for goal fields and old time/round fallback.
- Goal confirmation tests with fake `codex` executable for complete, incomplete, failures, timeout, invalid JSON, schema mismatch, precedence, and JSONL evidence.
- Hook/config tests for configurable Stop hook timeout, default bundled plugin timeout, preserving unrelated hooks/config, and installer parity.
- `pre_loop_continue` compatibility tests for incomplete goal, warnings, completion, and rapid-stop cut-short.
- Final verification: update docs, run `make fmt`, `go vet ./...`, and `make verify`.

## Assumptions And Defaults

- `goal` is an exclusive limiter; no `goal + min` or `goal + rounds` combined mode in this version.
- Default goal confirmation model is `gpt-5.5`.
- Default goal confirmation reasoning effort is `high`.
- Default managed `Stop` hook timeout is `2700` seconds.
- Default nested goal confirmation timeout is `2400` seconds.
- Model and reasoning are separate because Codex documents `model` and `model_reasoning_effort` as separate config fields.
- `xhigh` is model-dependent; codex-loop validates the spelling but lets Codex decide whether the selected model supports it.
- On confirmation failure or timeout, the loop continues with a warning because completion was not independently confirmed.
- The goal reviewer runs read-only and should not mutate the workspace.
